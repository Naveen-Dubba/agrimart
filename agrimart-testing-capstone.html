View Results Learn More

Project Overview

Agrimart is a vital digital bridge connecting farmers with agricultural buyers. Our capstone project systematically evaluated the platform to ensure a seamless, secure, and robust user experience. The testing scope covered four core pillars:

🛒

Functional Testing

Validation of critical user workflows including authentication, cart management, checkout processes, and order tracking.

⚡

Performance Testing

Evaluation of page load times, response under peak load, and system stability with simulated concurrent users.

🔒

Security Testing

Analysis of data encryption, SQL injection vulnerabilities, authorization controls, and secure payment processing.

📱

Usability & Responsiveness

Assessment of interface intuitiveness, navigation efficiency, and cross-device mobile compatibility.

Study Objectives

  • ✓ Validate Core Transactions: Ensure seamless execution of the end-to-end purchasing journey from product search to payment confirmation.
  • ✓ Identify Performance Bottlenecks: Measure system responsiveness and determine capacity limits to guarantee uptime during harvest seasons.
  • ✓ Assess Security Posture: Detect and mitigate vulnerabilities to protect sensitive user data and financial transactions.
  • ✓ Enhance User Experience: Evaluate interface clarity and accessibility to ensure the platform is intuitive for diverse user demographics.

Conclusion & Recommendations

Key Findings

  • The core functional modules (authentication, inventory, and payment) are robust and highly reliable.
  • Security posture is strong; no critical data exposure vulnerabilities were discovered.
  • The system successfully handles typical daily traffic loads without performance degradation.
  • A performance bottleneck was identified in the database search indexing during high-load simulations.

Recommendations

  • Database Optimization: Implement caching (e.g., Redis) and optimize search queries to resolve the stress test failure.
  • UI Polish: Address minor frontend state issues in the shopping cart when users rapidly update quantities.
  • Automation: Integrate Selenium test suites into the CI/CD pipeline to ensure regression testing on every deployment.
  • Accessibility: Improve screen reader compatibility on the checkout forms to meet full WCAG AA standards.
